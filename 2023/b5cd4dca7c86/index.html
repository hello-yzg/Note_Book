2. 数据类型

分配内存空间的关键字

2.1 整型

数据类型 占用空间 取值范围
short，短整型 2B -2^15~2^15-1
int，整型 4B -2^31~2^31-1
long，长整型 4B (Windows，Linux-32位)， 8B（Linux-64位） -2^31~2^31-1

6. 函数

6.1 概述

将一段经常使用的代码封装起来，减少重复代码——程序块

6.2 函数的定义

函数的定义一般分为五部分：

  1. 返回值类型
  2. 函数名
  3. 参数声明列表——形参
  4. 函数体语句
  5. return表达式

6.3 函数的调用

传入具体的参数——实参

调用时，实参的值会传递给形参

6.4 值传递

函数调用时，实参的值会传递给形参，并不受形参的影响

6.5 函数的常见样式

常见的函数样式有四种：

  1. 无参无返
  2. 有参无返
  3. 无参有返
  4. 有参有返

6.6 函数的声明

向编译器声明函数信息（一般情况下代码逐行执行，意味着函数定义必须写于调用之前，为了改善这种强秩序，引入声明机制）

函数的实际主体可以单独定义，但必须唯一

6.7 函数的分文件编写

让代码结构更加清晰，一般分为四个步骤：

  1. 创建后缀为.hpp的头文件
  2. 创建后缀为.cpp的源文件
  3. 在头文件中写函数的声明
  4. 在源文件中写函数的定义

通过 #include ”……“ 包含自定义的头文件

可以通过指针间接访问内存

  • 内存编号从0开始记录，为十六进制数字
  • 指针变量用于保存地址
  • 指针本质上是一个对象

7.3 指针所占内存空间

32位操作系统下：占用4字节内存空间

64位操作系统下：占用8字节内存空间

7.4 空指针和野指针

  • 空指针：指针变量指向内存中编号为0的空间——指针初始状态

空指针指向的内存不可访问（0~255内存编号由系统使用）

  • 野指针：指针变量指向非法空间
  1. 指针未初始化
  2. 指针局部初始化

野指针指向内存不可访问（无访问权限）

7.5 const修饰指针

const关键字 修饰变量

*运算符 对右侧变量 (地址) 取值；为右侧变量赋左侧类型的地址——指针声明)

  1. const修饰指针：常量指针（底层const）
  1. const修饰常量：指针常量（顶层const）
  1. const既修饰指针又修饰常量

==思考：*运算符本质为重载==

  1. ==左侧为类型，右侧为变量时：指针声明，将左侧默认变量取地址赋给指针变量==

  2. ==仅右侧为变量时：解地址==
