递归是什么

引用来自知乎作者 [Fireman A] 针对汉诺塔问题的回答，包含着对递归的深刻理解，同时充满悬疑短篇的猎奇感：

你来到某个热带丛林，意外发现了十层之高的汉诺塔。正当你苦苦思索如何搬动它时，林中出来一个土著，毛遂自荐要帮你搬塔。他名叫二傻，戴着一个草帽，草帽上有一个2字，号称会把一到二号盘搬到任意柱。

你灵机一动，问道：“你该不会有个兄弟叫三傻吧？”

“对对，哥你咋知道的？他会搬一到三号盘。“

”那你去把他叫来，我不需要你了。“

于是三傻来了，他也带着个草帽，上面有个3字。

你说：”三傻，你帮我把头三个盘子移到 C吧。“

三傻沉吟了一会，走进树林，你听见他大叫：”二傻，出来帮我把头两个盘子搬到 C !“

由于天气炎热你开始打瞌睡。朦胧中你没看见二傻是怎么工作的，二傻干完以后，走入林中大叫一声：“老三，我干完了!”

三傻出来，把三号盘从 A 搬到 B，然后又去叫二傻：“老二，帮我把头两个盘子搬回 A !”

三傻其实只搬三号盘，其他叫二傻出来干。最后一步是三傻把三号盘搬到 C，然后呼叫二傻来把头两个盘子搬回 C。

事情完了之后你把三傻叫来，对他说：“其实你不知道怎么具体一步一步把三个盘子搬到 C，是吧？”

三傻不解地说：“我不是把任务干完了？”

你说：“可你其实你兄弟二傻干了大部分工作呀？”

三傻说：“他不是我喊来的吗？”

你问到：“那二傻是不是也喊了别人？“

三傻笑了：“这关我什么事？”

你苦苦思索了一夜，第二天，你走入林中大叫：“十傻，你在哪？”

一个头上带着10号草帽的人，十傻，应声而出：“哥，你有什么事？”

“我要你帮把1到10号盘子搬到 C。“

“好的，老爷。“十傻转身就向林内走。

“慢着，你该不是回去叫你兄弟九傻吧？“

“你怎么知道的？“

“所以你使唤他把头九个盘子搬过来搬过去，你只要搬几次十号盘就好了，对吗？“

“对呀！“

“你知不知道他是怎么干的？“

“这关我什么事？“

你叹了一口气，决定放弃追问。十傻开始干活。树林里充满了此起彼伏的叫声：

“九傻，来一下！“

“老八，到你了！“

“五傻！。。。“

”三傻！。。。“

”大傻！“

你注意到大傻从不叫人，但是大傻的工作也最简单，他只是把一号盘搬来搬去。一段时间后，工作结束了。十傻来到你面前。

你问十傻：“是谁教给你们这么干活的？“

十傻说：“我爸爸。他给我留了这张纸条。”

他从口袋里掏出一张小纸条，上面写着：“照你帽子的号码搬盘子到目标柱。如果有盘子压住你，叫你上面一位哥哥把他搬走。如果有盘子占住你要去的柱子，叫你哥哥把它搬到不碍事的地方。等你的盘子搬到了目标，告诉你上面一位哥哥你做完了。“

你不解地问：“那大傻没有哥哥怎么办？“

十傻笑了：“他只管一号盘，所以永远不会碰到这种情况。”

但这时他突然变了颜色，好像泄漏了巨大的机密。他惊慌地看了你一眼，飞快地逃入树林。第二天，你到树林里去搜寻这十兄弟。他们已经不知去向。你找到了一个小屋，只容一个人居住，但是屋里有十顶草帽，写着一到十号的号码。

其实，递归之所以不能被我们很轻易的理解，是因为它的本质是一种”顶层设计“，这种自上而下的思维与人的正向思维相悖。我们一般的思维是自下而上的，是递推的，比如我们学习一篇课文，先要学会字，然后是词，再是句，最后成段，从基础到进阶，这是最有安全感的，因为每一步我们都清晰可控，我们花了大量的时间去经营过程，这是本能的思维，就像计算机先出现面向过程编程。

但当问题达到一定规模，或者我们拥有了大量递推经验时，再次面对同类问题时，我们又会本能地厌倦了循环往复的推演，有没有一种方式可以跳过这些过程呢？于是，我们开始偷懒，当一个规模为 n 的问题出现时，n-1 成为了偷懒的目标，发问：

如果 n-1 规模可以被解决，那么 n 规模是否同理？

从这里可以看出，递归是有它的特征的，即规模本身存在着一定的逻辑规律，不然怎么满足”同理“？换句话时，每一级规模都可以通过同一个变换得到，最常见的数学模型就是等差或等比队列。

那么，得到了肯定的答案后，结合代数思维，我们开始对问题进行降级，这个过程中，我们需要有一个定则：对数学归纳法的信任。为什么这么说，只有信任，我们才能不断对问题进行分解，否则分解到一半，可能就会怀疑，这中间是怎么实现的，我连怎么实现的都不知道，我怎么知道结果是不是正确的？自此我们刚刚建立的顶层思维将坍塌，重新回到解决问题的起点。递归之所以很难被人理解的桎梏就在这里，它比面向对象编程更加让我们感到没有安全感，但本质上，是我们脱离”亲力亲为”之后的滞空感。

只有我们信任身后的一切，才能轻松向后倒下，递易，归难。

花了大篇幅来讲递归的思想，可能会有人觉得华而不实，不如把时间花在一些递归的实际算法场景上，不然实际应用完全不知道怎么写递归，什么时候写递归，说实话，我也不全知道，但我在回顾一些旧的递归算法时，多了清晰感，不再那么有距离，那么严肃，对我来讲，这是一个好迹象。
